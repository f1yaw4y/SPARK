1. EXECUTIVE SUMMARY
================================================================================

The SPARK mesh network simulation validated onion-encrypted, address-based
routing across networks of up to 5,000 virtual nodes under realistic adverse
conditions.  The final routing architecture achieves 100% message delivery
across all tested configurations, including:

  - 5,000 nodes with 5% packet loss, node death, and temporary dropouts
  - Severe geographic bottlenecks (6-node bridge connecting two clusters)
  - Congestion queuing with LoRa bandwidth constraints (2 TX/tick per node)

No messages were lost in the final configuration.  Messages delayed by
congestion were queued and delivered (FIFO, first-come-first-serve), with
the longest queued message taking ~400,000 ticks through a 6-node bottleneck.

2. SIMULATION ARCHITECTURE
================================================================================

Routing Protocol:
  - 128-bit Node IDs derived from BLAKE2b hash of Ed25519 public keys
  - XOR-distance greedy forwarding (Kademlia-inspired), O(log N) convergence
  - 3-layer onion encryption (real crypto: X25519 + XChaCha20-Poly1305)
  - Gossip-based routing table population via beacons
  - No node has global knowledge; each knows ~140 peers via gossip

Reliability Mechanisms (final configuration):
  - Receipt ACK per hop (lightweight packet loss recovery)
  - Multi-path sending (2 copies per attempt via different gateways)
  - End-to-end retry with gateway exclusion (up to 6 attempts)
  - Weighted-random exploratory forwarding at dead-ends
  - FIFO TX queue with congestion-aware ACK timeouts (bottleneck mode)

Simulation Parameters:
  - Radio range:        1,500m base (LoRa SX1262 @ SF7/125kHz)
  - Max packet size:    255 bytes
  - K-bucket size:      40 entries per bucket
  - Max gossip:         6 entries per beacon
  - Beacon cycles:      20-25 (discovery phase)
  - Seed:               42 (reproducible)

Realism Features:
  - Packet loss:        5% per radio delivery (interference/collisions)
  - Range variance:     +/-20% per node (terrain/antenna variation)
  - Node death:         1-2% permanent failure during messaging
  - Temporary dropout:  0.2-0.3% per tick per node (brief glitch)

5. RELIABILITY MECHANISM EVOLUTION
================================================================================

The routing reliability was developed iteratively.  Each mechanism was
tested and the results drove the next improvement.

  Mechanism                         5,000-node delivery    Notes
  --------------------------------  --------------------   -----------------------
  Greedy XOR only                   ~35%                   Dead-ends, no recovery
  + DFS backtracking                ~70-84%                Visited set too large
  + Receipt ACK (packet loss)       ~91%                   Handles 5% loss well
  + E2E retry (3 attempts)          91.5%                  Same gateways = same path
  + Gateway exclusion on retry      99.3% (2k nodes)       Different gateways work
  + Multi-path (2 copies)           94.0%                  Deterministic dead-ends
  + Weighted-random exploratory     100.0%                 Path diversity solves it
  + FIFO TX queue (bottleneck)      100.0%                 Congestion -> delay, not loss

6. KEY FINDINGS
================================================================================

6.1 XOR Routing Scales Logarithmically
  - 200 nodes:    avg 29 ticks,  ~3-4 XOR hops
  - 2,000 nodes:  avg 253 ticks, ~8-10 XOR hops
  - 5,000 nodes:  avg 468 ticks, ~12-15 XOR hops
  - Consistent with O(log N) theoretical expectation

6.2 Gossip Populates Routing Tables Efficiently
  - After 25 beacon cycles, each node knows ~140 peers from ~100 regions
  - This is sufficient for O(log N) routing in a 128-bit address space
  - Coverage reaches 100% (every node is known by at least one other node)
  - Routing table size plateaus by cycle 20 (stable after ~15 cycles)

6.3 Multi-Path is Critical for Large Networks
  - Single-path delivery at 5,000 nodes: ~91%
  - Dual-path delivery at 5,000 nodes: ~94% (deterministic)
  - Dual-path + random exploration: 100%
  - Redundant paths provide the safety margin for dead-end recovery

6.4 Weighted-Random Exploration Solves Structural Dead-Ends
  - Pure greedy XOR hits local minima in large XOR address spaces
  - Deterministic "closest remaining neighbour" fallback always hits
    the same dead-end across retries
  - Exponentially-weighted random selection (closer = more likely, but
    not guaranteed) ensures different copies explore different routes
  - This is the single change that pushed delivery from 94% to 100%

6.5 Congestion Queuing Handles Bottlenecks Gracefully
  - TX rate limit (2/tick) simulates real LoRa bandwidth constraints
  - FIFO queue ensures first-come-first-serve fairness
  - ACKs bypass the queue (tiny, critical for reliability)
  - Queue-aware timeouts prevent premature retransmissions
  - Messages are delayed at bottlenecks, never rejected
  - Worst case: 400,337 ticks through a 6-node bridge -- delivered

6.6 Onion Encryption Works at Scale
  - 3-layer onion encryption applied to all 5,000-node test messages
  - Real Ed25519/X25519 keys, XChaCha20-Poly1305 encryption
  - No node sees both sender and recipient
  - Encryption overhead is negligible vs. routing latency

6.7 Network Resilience Under Adverse Conditions
  - 2% node death (100 out of 5,000 nodes killed): still 100% delivery
  - 5% packet loss per hop: handled by receipt ACK retransmission
  - +/-20% radio range variance: creates uneven topology, no impact
  - 0.3%/tick dropout rate (172k events in one test): no impact
  - All conditions applied simultaneously: still 100% delivery
